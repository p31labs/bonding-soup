Family layer · K₄,₄,₄ triadic cover

The agent tetrahedron sits above your personal tetrahedron, which sits above the family cage. Every family vertex pairs with one personal dock and (transitively) with one agent guardian. Children's vertices route to scribe through the child-mesh-unlock gate so phos-companion stays gated.

  • W (operator)structure → forge. Make.
  • S.J. (child)rhythm → scholar. Understand. child-gated
  • W.J. (child)creation → scribe. Remember. child-gated
  • C (co-parent)connection → counsel. Protect.

The six K₄ edges

Every K₄ vertex pairs with every other. These are the cooperation paths the hubs use when one ask reaches across two verbs.

  • forge ↔ counselSafe to ship. Build it; what could be sued over?
  • forge ↔ scholarDesign from research. Read these papers; scaffold what they imply.
  • forge ↔ scribeWrite what you build. Assemble the runbook from this PR's diff.
  • counsel ↔ scholarBrief the brief. What does the case law say about the situation?
  • counsel ↔ scribeFile the protect. Log the letter, classify the voltage, store the response.
  • scholar ↔ scribePublish the understanding. Take the synthesis; turn it into a record.

Dock your personal tetrahedron

One POST. The hub returns a 24h sessionId and four hub baseUrls. Use the sessionId via Authorization: Bearer … on every subsequent call. Pass publicKey · ts · nonce · sig for an Ed25519-signed dock; per-call signed envelopes are required when the session is signed. 

curl -X POST https://k4-agent-hub.trimtab-signal.workers.dev/v1/dock \
  -H 'content-type: application/json' \
  -d '{
    "clientId": "your-personal-tetra-uuid",
    "personalTetra": {
      "schema": "p31.personalTetra/1.0.0",
      "docks": { "structure": {}, "connection": {}, "rhythm": {}, "creation": {} }
    },
    "capabilities": ["ts-worker", "voltage-triage", "passport-mirror"],
    "publicKey": "",
    "ts": 1760000000000,
    "nonce": "abc12345",
    "sig": ""
  }'

Federation · multiple operator instances

Run another K₄ agent tetrahedron next to yours and let them peer. POST /v1/federation/peer with a signed instanceId; GET /v1/federation returns the aggregated topology (your local hub + every cached peer manifest). 

curl -X POST https://k4-agent-hub.trimtab-signal.workers.dev/v1/federation/peer \
  -H 'content-type: application/json' \
  -d '{
    "instanceId":"alpha-cluster",
    "manifestUrl":"https://other-hub.example/v1/manifest",
    "publicKey":"<base64url 32-byte Ed25519 pub>",
    "ts":1760000000000,
    "sig":"<base64url Ed25519 signature>"
  }'
