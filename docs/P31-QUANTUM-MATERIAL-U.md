## 1. Why "quantum" + "material U"

**Material U** here is *Material 3 grammar* — surface elevation, state layers, shape, motion, FAB / chip / button variants — but **personalized** by the operator's Cognitive Passport, not by image-derived dynamic color. Material 3's "you" is the person; P31's "U" is the operator who already told us how they want light, contrast, density, and motion.

**Quantum** is the K₄ refraction. The five P31 brand anchors — `teal · coral · phosphorus · butter · lavender` — are the SIC-POVM vertices of the K₄ tetrahedron (`docs/SIC-POVM-K4-ARCHITECTURE.md`). Every tonal step is generated by **mixing** an anchor against `--p31-void` (hub) or `--p31-paper` (org). Every quantum surface is a calm projection of the same coherent field — same mesh, different altitude.

> One sky. Twenty-three windows. **One palette. Six tones. Five anchors. Twelve component classes.**

## 2. Hard guards (non-negotiable)

This refactor is **purely additive**. It must not — and does not:

- ❌ paint brand chroma at Layer 1 (Gray Rock first paint stays inert)
- ❌ self-start motion on load
- ❌ replace `.p31-card`, `.p31-btn`, `.p31-glass`, `.p31-mission-trio*`, `.p31-return-ribbon`
- ❌ touch the frozen `p31-shared-surface.css` canon
- ❌ break the `cognitive-passport/p31-style.css` mirror parity check
- ❌ produce a partial-clone failure (the verifier skips cleanly when CSS is absent)

It must — and does:

- ✅ default to Gray Rock until the operator interacts (`:hover` / `:focus-visible` / `:active`)
- ✅ honour `prefers-reduced-motion` and `:root[data-p31-motion="none"]`
- ✅ compose with passport `data-p31-{density,contrast,motion}`
- ✅ degrade gracefully on browsers without `color-mix()` (modern Safari/Firefox/Chromium support it; fallback is the underlying brand var)
- ✅ stay on the `verify:p31-style` mirror diff (every token rides the generated CSS)

## 4. Tone scale (how it works)

```css
/* hub (dark) — generator output */
--p31-tone-teal-0: color-mix(in srgb, var(--p31-teal) 6%, var(--p31-void));
--p31-tone-teal-1: color-mix(in srgb, var(--p31-teal) 12%, var(--p31-void));
--p31-tone-teal-2: color-mix(in srgb, var(--p31-teal) 22%, var(--p31-void));
--p31-tone-teal-3: color-mix(in srgb, var(--p31-teal) 36%, var(--p31-void));
--p31-tone-teal-4: color-mix(in srgb, var(--p31-teal) 54%, var(--p31-void));
--p31-tone-teal-5: color-mix(in srgb, var(--p31-teal) 78%, var(--p31-void));
```

```css
/* org (light) — same anchor, mixed with paper instead of void */
[data-p31-appearance="org"] {
  --p31-tone-teal-0: color-mix(in srgb, var(--p31-teal) 6%, var(--p31-paper));
  /* … */
}
```

Tone `0` reads as Gray Rock; tone `5` is the loudest a quantum surface should touch. Tones `1–2` are the working surfaces for cards, chips, and tonal buttons. Tones `3–5` are reserved for accent rails and focus emphasis.

## 6. Personalization (passport bridge)

Cognitive Passport prefs already drive `:root` attributes via `cognitive-passport/lib/p31-subject-prefs.js`. The quantum layer composes with them — no new mechanism, no new state:

| Passport pref | Quantum effect |
|---|---|
| `data-p31-density="compact"` | `.p31-q-card` padding drops to `--p31-space-4` |
| `data-p31-density="spacious"` | `.p31-q-card` padding rises to `--p31-space-8` |
| `data-p31-contrast="high"` / `"max"` | `.p31-q-card` border switches to `--p31-cloud` |
| `data-p31-motion="none"` | `.p31-q-atmosphere` glow forced off; transitions clamped (existing global rule) |
| `prefers-reduced-motion: reduce` | All `.p31-q-*` transitions disabled at the per-component media query |

The passport itself is the personalization channel. Material 3's "dynamic color" — derived from a wallpaper image — is **not** the model; the operator's stated needs are.

## 7. Migration (existing → quantum, optional)

You do not need to migrate. The Gray Rock → Alive layer remains the recommended baseline.

If a surface wants Material 3 expressive grammar — typically dashboards, command bars, or product cards — adopt by **adding** classes:

- Replace `.p31-card` → `.p31-q-card` for surfaces that need elevation/tone.
- Replace `.p31-btn` → `.p31-q-button` for surfaces that need state-layer feedback.
- Add `.p31-q-fab` to the **one** primary action (per Material 3: never two per view).
- Use `[data-p31-tone]` to emphasize K₄ vertex semantics: `coral` for warmth, `phosphorus` for confirmation, `butter` for soft attention, `lavender` for depth/wonder, default `teal` for trust.

`.p31-card` and `.p31-btn` are not removed; they remain canonical for Gray Rock surfaces.
