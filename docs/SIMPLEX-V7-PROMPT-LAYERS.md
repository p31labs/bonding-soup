# SIMPLEX v7 — five structural layers (Composer prompt grammar)

**Purpose:** These layers make “find first, build second” and **WCD + OQE** mechanically enforceable — not vibes.

| Layer | Job |
|--------|-----|
| **1 — Find first, build second** | Fetch live URLs, map repo trees, enumerate `wrangler.toml`, KV/D1 bindings **before** design. Subsequent steps reference discovered facts only. |
| **2 — Geometric grammar** | Tetrahedron → **K₄** → Jitterbug → Posner → Floating Neutral → Larmor. Binary tests: four vertices? every edge? **No hub. If designing Wye, stop.** |
| **3 — Constraint surface** | BONDING baseline **424/32** sacred; live sites immutable without intent; litigation unpublished without review; hostile mail never raw-delivered; **credentials never in source** (SOP-03). Rails, not hacks. |
| **4 — Ordered WCDs + explicit OQE** | Six packages (see **`docs/SIMPLEX-V7-COMPOSER-BRIEFING.md`**); each states **verifiable close condition** before work (e.g. D1 table count, route matrix). |
| **5 — Trimtab test** | If 50 lines do what 10 could, stop — find the slot in existing geometry. **Wire, don’t inflate.** |

Canonical implementation scaffold: **`simplex-v7/`** (repo root). Confidential operator briefing: **`docs/SIMPLEX-V7-COMPOSER-BRIEFING.md`**.
