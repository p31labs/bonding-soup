THE PHILOSOPHY

PHOS is not a navigation bar. PHOS is not a hamburger menu. PHOS is a voice-first, inference-driven routing system that reduces Shannon entropy through Bayesian questioning.

The core insight: most navigation fails because it presents too many choices upfront. Hick's Law tells us decision time grows logarithmically with choice count. PHOS inverts this — instead of showing all destinations, it asks 2-3 questions and routes efficiently.

Core Principle: Gray Rock by Default

PHOS surfaces are inert by default. No animation. No auto-play. No surprise modals. The operator must explicitly "wake" the interface through interaction. This respects:

  • Spoon budgets (cognitive energy reserves)
  • Attention volatility (AuDHD context switching costs)
  • Sensory sensitivity (autistic hyperacusis, photophobia)
  • Physiological limits (hypoparathyroidism calcium crashes)

THE BAYESIAN ROUTER

PHOS uses a 3-question decision tree with Shannon entropy reduction:

Q1: Are the children with you? → If YES: GARDEN, GEODESIC (child-safe surfaces)
Q2: Is there external deadline/voltage? → If YES: BUFFER, OPS, CORTEX (urgency surfaces)
Q3: Are we in 'Mechanic' mode? → If YES: OPS, VIBE, GEODESIC, CORTEX, BUILD

Each question reduces the candidate set by ~50%, achieving O(log n) routing vs O(n) scanning.

SAFE MODE PROTOCOL

PHOS implements the most aggressive safe mode in web UI:

CSS Animation Kill

All transitions set to 0s. Keyframe animations removed. Canvas opacity forced to 0.

WebGL Destruction

loseContext() called on all WebGL contexts. GPU threads released. Prevents calcium crash acceleration.

Color Contrast Maximum

Pure black (#000) background. Pure white (#fff) text. All decorative gradients disabled.

Motion Respect

Respects prefers-reduced-motion automatically. Safe mode toggle for manual override.
